Virtual Jaguar Rx: The Documentation

Foreword

Virtual Jaguar Rx is based on the original Virtual Jaguar, the Rx is mostly focus to integrate a new debugger or to improve the one already available, however such integration can lead to emulation improvement on the side. 'Rx' means 'Revision x', where x has no defined value.

System Requirements

Virtual Jaguar Rx should be able to run on most late model computers equipped with an OpenGL compliant video card. It requires a lot of raw oomph in its present form to run well; do not be surprised if it doesn’t run at 100% speed with all the options turned on. Certain subsystems of the Atari Jaguar are not optimized yet and as such take more time than they would if they were optimized.

In Depth

This version of Virtual Jaguar Rx follows the original Virtual Jaguar's GUI and as such should be fairly easy to use and understand. However, some options may not do what you expect: What follows is an exhaustive look at what the various toolbar buttons and menu options do.

The Toolbar

The main Virtual Jaguar Rx toolbar contains the following buttons:

  • Power
  • Pause
  • Frame Advance
  • Insert Cartridge
  • Use CD Unit
  • Screenshot
  • Scale x1
  • Scale x2
  • Scale x3
  • NTSC
  • PAL
  • Blur
  • Full Screen

Here’s what they do.

Power

This toolbar button acts just like the power switch on a real console, complete with untuned tank circuit noise showing on the screen when the emulated Atari Jaguar is turned off. Some options can only be changed after the power has been turned off, such as the NTSC/PAL switches, or the “Use CD Unit” option. The BIOS selection in the "Configure" settings will only take effect after the power has been cycled.

Pause

Pressing the Pause toolbar button will pause execution of the emulated Atari Jaguar. Note that most games have a pause feature that is built into the game; this feature is separate from and will override any built-in game pause feature. Also, the emulator will go into pause mode if the power is on when you activate the “Insert Cartridge” dialogue. Pressing the Esc key will also activate this option. Selecting this option in any of the ways listed above, while in pause mode, will take it out of pause mode. This feature has also a related key binding.

Frame Advance

Pressing the button causes the emulator to run for one frame (1/60th of a second for NTSC, 1/50th for PAL), and update the screen. This function can also be activated by using his keybinding on your keyboard. This option only works when Virtual Jaguar Rx is paused.

Insert Cartridge

Pressing the Insert Cartridge button will bring up the Insert Cartridge dialogue; it can also be launched by pressing his associated keybinding on your keyboard. On the left side of the dialogue will be a list of software that Virtual Jaguar Rx recognizes from the directory you tell it to look into (see the “Configure” menu option and the “General” tab, “Software” item for more information). Selecting an item in the list will bring up some information about the software on the right side of the dialogue if the emulator knows about it. Pressing the icon in the lower right-hand side of the dialogue will then cause Virtual Jaguar Rx to attempt to run the selected item. Emulator cycles the power when it attempts to run the software you select.

There are also some keyboard shortcuts that come with the Insert Cartridge dialogue: Pressing a letter key will highlight the nearest item in the list that starts with that letter. Pressing more than one letter key in quick succession will narrow in on the item that matches those letter keys pressed. Pressing the Esc key will dismiss the dialogue without attempting to load anything, and pressing the Enter key is the same as pressing the icon on the lower right-hand side of the dialogue.

Container & ROM image

If you’re wondering why some cartridges have labels in the “Insert Cartridge...” dialogue and some don’t, read on...

Virtual Jaguar Rx looks at your ZIP file as a container—if it finds what it expects in the ZIP, it will use it. For example, label images: If you put an image file of a cartridge label in the ZIP file with your ROM image, the emulator will use that image in the “Insert Cartridge...” dialogue — assuming it’s a valid image file. The image should be 365 x 168 for it to show up on the big cartridge image without any scaling; any bigger is just a waste of space and Virtual Jaguar Rx will scale it down to 365 x 168 anyway.

Using the ZIP file as a container this way opens up some exciting possibilities: Box art, overlays and documentation can be included and future versions of Virtual Jaguar Rx will be able to utilize them.

Use CD Unit

Pressing this toolbar button will virtually plug in a Atari Jaguar CD unit into the emulated console. It can only be toggled with the virtual machine powered off.

As of the release date for this emulator, this option DOES NOT WORK 100% the way it’s intended to: When it’s working correctly, selecting this option will allow you to insert a Atari Jaguar CD game into your computer’s CD/DVD drive and play it as if it were on a real console system. For now, you can see the CD BIOS make pretty patterns on your screen if you use it.

Note that once this is working as it should, the DSP will be required to be enabled for it to work properly. It WILL NOT WORK properly without the DSP.

Screenshot

Pressing the screenshot button will create a picture file based on the screen displayed in the main window. The file will be saved in the screenshot dedicated directory. This feature has also an associated key binding.

Scale x1

Selecting this toolbar button will cause the screen to be displayed with pixels that are sized 1 to 1 with your display; this will give you a very small display.

Scale x2

This will cause the screen to be displayed with pixels that are sized 2 to 1 with your display.

Scale x3

This will cause the screen to be displayed with pixels that are sized 3 to 1 with your display; this should yield a fairly large display.

NTSC

Selecting this item will cause the emulator to behave as if it were an NTSC type console running at 60 Hz. This option can only be toggled when the virtual machine is powered off.

PAL

Selecting this item will cause theemulator to behave as if it were a PAL type console running at 50 Hz. As an added bonus, you also get a few extra scan lines as compared with NTSC mode; some games will even utilize this extra screen real estate. This option can only be toggled when the virtual machine is powered off.

Blur

Normally, the emulated Atari Jaguar’s display shows pixels that are razor sharp and is what you would likely see if the Atari Jaguar was hooked up to a high-quality monitor. Selecting this item will cause the emulator’s display to be averaged, resulting in a screen that looks slightly blurry—more like what you would expect to see if the console was hooked up to a television. This is an aesthetic option that some people prefer; but you may or may not like it.

Full Screen

Selecting this toolbar button will switch the display into full-screen mode; selecting it again will switch it back to windowed mode. This option has also an associated keybinding. Note that on monitors with a wide aspect ratio that Virtual Jaguar Rx will still display its virtual screen in a 4:3 aspect ratio—this is normal and expected.

Jaguar Menu

This menu has the following options:

  • Power
  • Pause
  • Insert Cartridge...
  • Use CD Unit
  • Configure
  • Status
  • Exit

Power, Pause, Insert Cartridge... and Use CD Unit do the same as the toolbar buttons previously described.

Configure launches the Configure dialogue. A full description is given in the “Configure Dialog” section below.

Status launches a window with emulator internal information. It includes a couple of M68000 cycles counters. Feature can also be launch using his key binding.

Exit quits Virtual Jaguar Rx immediately. You can also quit the emulator by using his keybinding.

Help Menu

This menu has the following options:

  • Contents
  • About

The Contents is what you’re reading right now. This dialogue can be dismissed by clicking on the “X” in the upper corner of the dialogue, or by pressing the Esc key on your keyboard.

The About option shows a credits dialogue. It can be dismissed by clicking on the “X” in the upper corner of the dialogue, or by pressing the Esc key on your keyboard.

Tools Menu

This menu has the following options:

  • Dump Save State...
  • Load Save State...
  • Save Slot 0... to 9...

The Dump Save State... option save the current state of the emulated console (memory, HW registers, etc.). Feature can also be called using his key binding. The dump is performed only if a cartridge has been inserted.

The Load Save State... option load a dump save state, his associated cartridge must be loaded first. Feature can also be called using his key binding.

The Save Slot 0... to 9... selects one of the save state to be used by the Dump and Load Save State feature. The option has also a key binding.

The Configure Dialog

The Configure dialogue has the following tabs:

  • General
  • Models and BIOS

General Tab

Under the General tab, there are the following options:

  • EEPROMs
  • Software
  • Screenshots
  • Save States
  • Enable GPU
  • Enable DSP
  • Start Virtual Jaguar in full screen
  • Show all files in file chooser
  • Use fast blitter

EEPROMs

This field holds the location on your filesystem where your EEPROM files will live (see the “EEPROMs” section below for details). The path can be either absolute or relative; if relative, it will be relative to where the Virtual Jaguar Rx executable is located on your filesystem.

Software

This field holds the location on your filesystem where your Atari Jaguar software files will live (see the “Software” section below for details). The path can be either absolute or relative; if relative, it will be relative to where the Virtual Jaguar Rx executable is located on your filesystem.

Screenshots

This field holds the location on your filesystem where your screenshots files will live (see the “Screenshots” section below for details). The path can be either absolute or relative; if relative, it will be relative to where the Virtual Jaguar Rx executable is located on your filesystem.

Save States

This field holds the location on your filesystem where your save states files will live (see the “Save States” section below for details). The path can be either absolute or relative; if relative, it will be relative to where the Virtual Jaguar Rx executable is located on your filesystem.

Enable GPU

The GPU is one of the three main processors in the Atari Jaguar, and checking this option enables it. Most software will not work without this.

Enable DSP

The DSP is one of the three main processors in the Atari Jaguar, and checking this option enables it. Much software will work without it, however, there is some software that will not. Note that if the DSP is disabled, there will be no sounds.

Start Virtual Jaguar in full screen

Emulator will display his video output in full screen.

Show all files in file chooser

Normally the “Insert Cartridge...” dialogue will ignore any files it encounters that it doesn’t know how to deal with. You can override this behavior by checking this option. However, this will not cause Virtual Jaguar Rx to be able to load such files.

Use fast blitter

Checking this option will force the emulator to use the older, less compatible yet faster blitter. Some games will not work properly with this option checked.

Models and Bios Tab

Under the Models and Bios tab, there are the following options:

  • Atari Jaguar model
  • Retail BIOS
  • Developer BIOS

Atari Jaguar Model

Option to describe the Atari Jaguar model; set as "M" by default; there is no model specific usage except when selecting the retail BIOS.

Retail BIOS

Selecting this option will cause the emulator to run with the built-in system BIOS upon powering on. This is usually not necessary, but some games will not run correctly without it. Note that this option will only affect cartridge images; also, it won't take effect until the virtual machine has been power cycled.

Developer BIOS

Selecting this option will cause the emulator to run with the built-in developer BIOS upon powering on. This option has been set mostly for testing purposes. Note that this option will only affect cartridge images; also, it won't take effect until the virtual machine has been power cycled.

Exceptions Tab

Under the Exceptions tab, there are the following options:

  • Allow writes to cartdridge ROM
  • Allow M68000 exception catch
  • Allow writes to unknown memory location

Allow writes to cartdridge ROM is the option allowing or not the cartdridge ROM writing. If allowed, emulator won't interfere but when turned off the emulator will display a dialog box with proper information, at this point user can allow to continue or stop the code.

Allow M68000 exception catch is the option allowing or not the exception catch from the M68000. If allowed, the emulator will display a dialog box with proper information and user will have the choice to continue or to stop the code.

Allow writes to unknown memory location is the option allowing or not the exception catch from writing to an unknown memory location. If not allowed, the emulator will display a dialog box with proper information and will stop the code.

Controllers Tab

Under the Controller tab, there are the following options:

  • A combobox labeled “Host Device”
  • A combobox labeled “Map Name”
  • Two buttons labelled “+” and “-”
  • A combobox labeled “Maps to”
  • A picture of a Jaguar controller showing which inputs are defined
  • A button labeled “Define All Inputs”

Virtual Jaguar Rx utilizes a profile system for controllers. What this means is that you can set up any device that the emulator recognizes with any number of different configurations for it. Here’s how it works:

The “Host Device” combobox lets you select whichever gamepad device you have connected to your computer. This, plus the unique “Map Name” will show the keymapping set up for that particular host device/key map combination.

The “Map Name” combobox lets you set up multiple key maps for your detected device. To add a new key map to the current device, press the “+” button next to the combobox. It will then ask your for a human readable name for the new map. If you want to remove any maps that you don't want anymore, you can delete the currently selected map by pressing the “-” button; it will ask for confirmation. Keep in mind that this action cannot be undone!

The “Maps to” combobox tells Virtual Jaguar Rx which virtual controller slot you would like your device to be plugged into. Selecting None disables that key mapping for that device. Selecting Controller #1 will map the device to virtual controller slot #1. Selecting Controller #2 will map the device to virtual controller slot #2. Selecting Either one that’s free will attempt to connect the device to a free virtual controller slot if one is still available after all other connected devices have had a chance to plug into a definite slot. If other devices have filled up the virtual slots, this option will cause the device to not attempt to connect itself.

Hovering over a letter on the picture of the Atari Jaguar controller will highlight it; clicking on the letter will let you redefine that input by pressing a key on the keyboard or button on a connected gamepad. If you prefer, you can define all inputs at once by clicking on the “Define All Inputs” button. It will then step through each input on the controller, one at a time, and wait for you to press the key/button that you wish to set it to. You can exit the key redefinition process at any time by pressing the Esc key on your keyboard.

NOTE: For gamepads to be detected, they must be connected to the computer before starting Virtual Jaguar Rx.

Key Bindings Tab

This tab adapts based on the stock/retail, alpine and debugger mode. User can change the emulator keyboards interactions but not all features have keybinding.

EEPROMs

EEPROMs (Electrically Erasable Programmable Read Only Memory) were typically installed on Atari Jaguar cartridges to save high scores and the like, and Virtual Jaguar Rx emulates this. The EEPROMs are keyed to specific games by making a CRC32 checksum of the game and using this as a filename. If an EEPROM file does not exist, it is created when the emulator writes to it. The EEPROM files are stored in the location specified by the “EEPROMs” field of the Configuration dialog, under the “General” tab.

Software

Software can be either uncompressed or stored in a ZIP file. Virtual Jaguar Rx looks for software in the location specified by the “Software” field in the Configuration dialog, under the “General” tab.

The emulator is a bit picky about file formats, but tries its best to figure out what you’re trying to feed it. In general, if the file is a ROM image, or an Alpine image, Virtual Jaguar Rx will recognize and utilize it. If it’s a file meant to load into the emulated Atari Jaguar’s RAM space, Virtual Jaguar Rx will load and run it as long as the file contains proper headers. A headerless file of this kind (as opposed to a cartridge or Alpine ROM image) is useless; it contains no information to help the emulator knows how to load and run it. There is no excuse for this kind of file to exist other than programmer's own decision.

Screenshots

Some Issues You May Encounter

Most of the time Virtual Jaguar Rx tries to do the right thing with what you tell it to do. There are some options, however, that would be more annoying to use if Virtual Jaguar Rx got in the way and forced you to do the right thing, so, in those places where it’s possible to tell the emulator to do bad things, it will willingly oblige.

Some cartridge based games will not work without either the DSP enabled, the BIOS enabled, or both. Virtual Jaguar Rx will let you attempt to run those games with those options disabled, but the result probably won’t be what you want.

Jaguar Programmer’s Zone

Virtual Jaguar Rx has 2 modes available that makes it more convenient for developers to use. In order to use a mode, you must start the emulator with the --alpine or --debugger option. Using the debugger mode will also active the alpine mode.

The options will only work when Virtual Jaguar Rx is run in Alpine or Debugger mode, that is, when the --alpine or --debugger flag is passed in. If you do not pass such flag, Virtual Jaguar Rx will run as it normally does, i.e., as a stock/retail Atari Jaguar, and nothing will be loaded by default.

Alpine Mode

Configuration

When in Alpine mode, a specific tab is made available in the configuration dialogue which gives you the following options:

  • ROM to load
  • ABS to load
  • Windows refresh

ROM to load is the path to a Atari Jaguar executable file that will be loaded into the cartridge memory space at $800000 (Alpine ROM images will load at $802000). As a convenience to coders, this file can be of any length, as long as it fits into the cartridge memory space (a file of this type is treated as an Alpine ROM and is loaded at $802000). The file thus loaded into memory will not be executed.

ABS to load is the path to a Atari Jaguar executable file that will be loaded in the main RAM. This file must have valid headers, otherwise the emulator won’t be able to load it. Assuming the file exists and has the correct headers in it, it will be loaded and run when Virtual Jaguar Rx is first run.

Windows refresh is the refresh rate for open windows. The smaller the rate, the faster the refresh window will occur. A 0 value will disable the refresh operations.

Memory Browser

This window lets you inspect Virtual Jaguar Rx’s emulated main memory space. The PgDn key will advance the top of the window’s memory location by 480 bytes, and PgUp will go back by the same amount. The Plus (+) key will advance the top of the window’s memory location by 16 bytes, and Minus (-) will go back by the same amount. Entering a memory location, in hexadecimal, in the edit field and pressing the Go button will display memory from that location. It can be dismissed by clicking on the “X” in the upper corner of the dialog, or by pressing the Esc key on your keyboard.

Stack Browser

This window lets you inspect Virtual Jaguar Rx’s emulated memory stack (SP) space. The PgDn key will advance the top of the window’s memory location by 480 bytes, and PgUp will go back by the same amount. The Plus (+) key will advance the top of the window’s memory location by 16 bytes, and Minus (-) will go back by the same amount. It can be dismissed by clicking on the “X” in the upper corner of the dialog, or by pressing the Esc key on your keyboard.

CPU Browser

This window shows the state of the various CPUs in the emulated system; a memory breakpoint (read & write) can also be set via the BPM setting. The window can be dismissed by clicking on the “X” in the upper corner of the dialog, or by pressing the Esc key on your keyboard.

OP Browser

This window shows the current state of object list that the Object Processor (OP) is pointing to. Note that most of the time, this will likely show an object list that has had its objects processed. It can be dismissed by clicking on the “X” in the upper corner of the dialog, or by pressing the Esc key on your keyboard.

M68K Disassembly Browser

This window provides a disassembly of 68000 opcodes. It can be dismissed by clicking on the “X” in the upper corner of the dialog, or by pressing the Esc key on your keyboard. Entering an address location, in hexadecimal, in the edit field and pressing the Go button will display disassembly from that location. 

RISC Disassembly Browser

This window provides a disassembly of RISC opcodes. It can be dismissed by clicking on the “X” in the upper corner of the dialog, or by pressing the Esc key on your keyboard. Entering an address location, in hexadecimal, in the edit field and pressing the Go button will display disassembly from that location. 

HW Registers Browser

This window provides a list of the hardware registers content; for the moment, only the blitter is displayed. It can be dismissed by clicking on the “X” in the upper corner of the dialog, or by pressing the Esc key on your keyboard.

ROM Cartridge Browser

This window lets you inspect the ROM cartridge space. A 6MB size is selected by defaut, but can be changed to 1MB, 2MB, and 4MB. The PgDn key will advance the top of the window’s memory location by 480 bytes, and PgUp will go back by the same amount. The Plus (+) key will advance the top of the window’s memory location by 16 bytes, and Minus (-) will go back by the same amount. Entering a memory location, in hexadecimal, in the edit field and pressing the Go button will display memory from that location. It can be dismissed by clicking on the “X” in the upper corner of the dialog, or by pressing the Esc key on your keyboard.

Debugger Mode

In the debugger mode, the main window doesn't display anymore the video output; it is replaced by a source code files viewer and a disassembly window made of tabs to allow tracing / display of code. 3 main components can be used.

Source code files browser is available for convenience reason; it display all the filenames found in the DWARF information. It is turned off by default but user can turn it on.

Sources tab is populated by C, or asm, source files following the M68000 program counter (PC). User can trace source code within the tab or navigate to look at the sources.

Disassembly tabs, each of them is dedicated to a specific processor (M68000, GPU, and DSP). Currently, only the M68000 is tracable, others 2 display only the code pointed by their respective program counter (PC).

Configuration

When in Debugger mode, a specific tab is made available in the configuration dialogue which gives you the following options:

  • Disassembly lines
  • Source file search paths
  • /cygdrive/ directory reference removal
  • Display M68000 opcodes
  • Display HW labels
  • Display source filename
  • ELF sections check

/cygdrive/ directory reference removal allows to handle or not sources compiled with a cygwin gcc M68K compiler. At the moment, the removal is done whatever the option is on or off.

Disassembly lines represent the number of disassembly lines to display in the M68000 tracing window. This doesn't include the number of source lines displayed.

Source file search paths may be used in case of the debugger cannot find files path in the DWARF information.

Display M68000 opcode allows or not the opcode display in the M68000 tracing window. Can be useful in case of user prefers to just see the disassembly.

Display HW labels allows or not the name of the Atari Jaguar's hardware registers as reported in the official documentation.

Display source filename allows or not the source code filename display depend the disassembly in the M68000 tracing window. If source is not found or outdated, a colored message will be shown.

ELF sections check allows or not potential unknown ELF section; it will stop, or not stop, the sections decoding.

Restart

Restart M68000 code (PC) and the stack (SP) pointer to the original ones provided in the ROM.

Step Into

Trace into M68000 instruction, similar to a step by step tracing.

Step Over

Trace over M68000 instruction such as jsr/bsr, otherwise this feature functions like a Step Into.

Breakpoints

List the breakpoints and their status. The first breakpoint represents the breakpoint set via the CPU browser, also named as BPM.

New Function Breakpoint

Add breakpoint to a function.

Delete All Breakpoints

Delete all breakpoints, also reset the BPM.

Disable All Breakpoints

Disable all breakpoints, and also disable BPM.

Exception Vector Table

Display a partial table of the M68000 exception vectors as reported by Motorola.

Output Video

Display the video output in a separate window.

All Watch

Display all global variables used by the Atari Jaguar code. It uses the available DWARF debug information. It has a search feature to look for a specific global variable.

Locals

Display the local variables of the traced M68000 function. It uses the available DWARF debug information, and the A6 register to point on the locals. A variable used as register will have his name in blue. An error icon will be displayed in case of the variable cannot be read, a warning icon will be displayed in case of a type is not supported.

Call Stack

Display the M68000 call stack functions. It uses the available DWARF debug information, and the A6 register to point on the stack.

Heap Allocator

Display the status of all memory allocations. It uses a specific structure memory allocation, if such structure is not found then nothing will be displayed.

Memory (1-4)

This 4 independent windows lets you inspect the emulated memory space. The PgDn key will advance the top of the window’s memory location by 480 bytes, and PgUp will go back by the same amount. The Plus (+) key will advance the top of the window’s memory location by 16 bytes, and Minus (-) will go back by the same amount. Entering a memory location, in hexadecimal, label or decimal, in the edit field and pressing the Go button will display memory from that location. It can be dismissed by clicking on the “X” in the upper corner of the dialog, or by pressing the Esc key on your keyboard.

Save Dump As...

Save a memory portion to a file. It is not related to the save states feature.

Cartridge directory and files list

List the cartridge's directory and files; available only if specific structure is used.
